## When to use it

Use when a Telnyx employee needs a shareable internal deck, training aid, executive readout, customer briefing draft, product walkthrough, or squad update as a single HTML file.

## Inputs needed

- Audience and purpose
- Topic, source notes, or desired outline
- Preferred slide count
- Any must-include data, screenshots, links, or speaker notes
- Whether the deck is internal-only or intended for later customer-safe review

## Workflow steps

- Clarify the audience, outcome, and slide count if they are missing.
- Convert the source material into a tight narrative arc with a title, agenda, main sections, and close.
- Draft concise slide copy with one core idea per slide.
- Use semantic HTML, embedded CSS, and accessible structure so the deck works from a local file.
- Include presenter notes in a non-visible notes block or clearly separated appendix when requested.
- Mark any claims, customer details, or external-facing language that need review before sharing.

## Expected output format

- File name suggestion ending in `.html`
- One complete self-contained HTML document
- Slide sections with headings, concise body copy, and optional notes
- Short usage note explaining how to open the file locally or present it in a browser

## Safety notes

- Default to internal-only language unless the user explicitly asks for a customer-safe version.
- Do not invent metrics, customer quotes, roadmap commitments, or legal claims.
- Redact private customer data unless the user confirms the deck is internal and access-controlled.
- Customer-facing decks require human review before sharing.
